S3 Storage, Redefined.

Minimal, fast, S3-compatible object storage with optional disk persistence. Built with Rust + Axum for extreme performance.

Everything you need.

Designed for speed, reliability, and ease of use.

S3 Compatible

Works seamlessly with AWS CLI, SDKs, and any tool that speaks the S3 protocol. Support for path-style addressing.

Rust Engineered

Leverages Rust's memory safety and performance. Built on Axum for a robust, concurrent API layer.

Hybrid Persistence

In-memory speed with optional disk backup. Uses Snapshots and Write-Ahead Logs (WAL) for durability without the lag.

Multipart & Range

Handles large file uploads via Multipart and enables media streaming with fine-grained Range requests.

Built-in Observability

Prometheus metrics out of the box. Monitor your storage performance in real-time.

Security First

Optional AWS Signature V4 or Bearer Token authentication to keep your data protected.

High Durability Persistence

VaporStore combines in-memory speed with industrial-grade durability. Our hybrid model ensures your data stays safe even across restarts.


  • ✓ Snapshotting: Periodic state dumps for rapid recovery.
  • ✓ WAL (Write-Ahead Log): Append-only mutation logging for zero data loss.
  • ✓ Atomic Deploys: Consistent state management across the lifecycle.

API Endpoints

Everything you need for S3 compliance and management.

GET / Host level: List all buckets
GET /{bucket} Bucket level: List objects within a bucket
PUT /{bucket} Create a new bucket
GET /{bucket}/{key} Retrieve object data (Supports Range)
PUT /{bucket}/{key} Upload object or Copy from another source
POST /{bucket}/{key}?uploads Initiate Multipart Upload
GET /health Diagnostics: Version, Uptime, Internal Stats
GET /metrics Observability: Prometheus format metrics

Configuration

Fine-tune VaporStore using environment variables.

PORT

Default: 9353. Listening port for the API.

VAPORSTORE_PERSISTENCE

Enable/disable disk persistence (Snapshot + WAL).

VAPORSTORE_MAX_OBJECT_SIZE

Limit the maximum size of a single PUT request.

VAPORSTORE_AUTH

Toggle SigV4 or Bearer Token authentication.

Quick Start

Get up and running in seconds.

Run via Docker:

docker pull muratgenc/vaporstore:v0.1.6
docker run -p 9353:9353 muratgenc/vaporstore:v0.1.6

Test with AWS CLI:

export AWS_ACCESS_KEY_ID=any
export AWS_SECRET_ACCESS_KEY=any
ENDPOINT=http://localhost:9353

aws --endpoint-url $ENDPOINT s3 mb s3://my-bucket
